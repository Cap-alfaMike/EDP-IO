EDP-IO Walkthrough

Summary

Built a mock production-ready Enterprise Data Platform for Retail/Azure demonstrating Tech Lead/Architect patterns.

File Structure (~50 files)

Core Utilities (src/utils/)

  • config.py - Pydantic Settings + feature flags
  • security.py - SecretProvider, PIIMasker
  • logging.py - Structured logging

Ingestion (src/ingestion/)

  • bronze_writer.py - Delta Lake MERGE
  • oracle_ingest.py - Oracle ERP
  • sqlserver_ingest.py - SQL Server
  • mock_data.py - Retail data generator
  • contracts.yaml - Data contracts

LLM Observability (src/observability/)

  • log_analyzer.py - Error analysis
  • schema_drift.py - Schema changes
  • doc_generator.py - Auto-docs
  • rag_context.py - Chained RAG
  • llm_metrics.py - Usage tracking

Orchestrator (src/orchestrator/)

  • dag_daily.py - Airflow DAG

dbt (dbt_project/models/)

Silver: stg_customers, stg_products, stg_orders (SCD2) Gold: dim_customer, dim_product, dim_date, fact_sales

Streamlit (app/)

  • main.py - Executive dashboard
  • 1_pipeline_status.py
  • 2_data_quality.py
  • 3_lineage.py
  • 4_ask_architect.py
  • 5_llm_observability.py - NEW

DataOps

  • main.tf - Azure resources
  • ci.yml - CI/CD
  • dbt-daily.yml - Scheduled runs

How to Run

pip install -r requirements.txt
streamlit run app/main.py

Production Deployment

  1. Set ENABLE_AZURE_INTEGRATION=true
  2. Configure Key Vault secrets
  3. Run terraform apply -var-file="environments/prod.tfvars"
